# RAPID (Rapid Aircraft Planform Iterative Design Tool)
**Developed by Ahmet Emre Arslan - Released under MIT License**

An NSGA-II and NASA OpenVSP-based optimization tool for aircraft wing planform geometry synthesis, with an integrated **XFOIL airfoil co-optimization** stage and support for both conventional and **blended-wing-body (tailless)** configurations. It couples an NSGA-II genetic optimizer with the OpenVSP 3.47.0 / VSPAERO panel solver and the XFOIL 6.99 viscous solver. The heavy binaries (Python runtime, OpenVSP, XFOIL) are installed locally in a short one-time setup — see **[SETUP.md](SETUP.md)**. 

*The default wing constraints and reference parameters are loosely based on the C-130 Hercules, but can be fully customized inside the source code.*

---

## 📂 Repository Structure

* **`RUN_OPTIMIZATION.bat`**: Main executable to launch the parallel optimization engine.
* **`CONFIGURE.bat`**: Interactive editor for constraints, design-variable bounds, and flight conditions (shows each value's default *and* your last-entered value).
* **`RUN_BWB.bat`**: Launches the optimizer in **BWB / tailless mode** using the ready `system_files/bwb_config.json` preset (static-margin stability + reflexed airfoils).
* **`RUN_PLOTTER.bat`**: Launches the standard visualization dashboard (Pareto front & wing geometries).
* **`RUN_PERFORMANCE_PLOTTER.bat`**: Launches the advanced interactive performance mapping dashboard (Range vs. Endurance).
* **`wing_models/`**: Holds generated wing models, aerodynamic solver logs, and the checkpoint file for pausing/resuming.
* **`PLOTS/`**: Stores the high-resolution graphical outputs generated by the plotters.
* **`RESET.bat`**: Safeguarded script to wipe previous results and reset the optimization study.
* **`system_files/xfoil_handler.py`**: Headless XFOIL 6.99 driver used by the airfoil co-optimization stage.
* **`system_files/xfoil/`**: Embedded portable XFOIL solver binaries.
* **`airfoil_database/`**: Local airfoil coordinate catalogs, grouped by Reynolds regime (`Low_Re`, `Medium_Re`, `High_Re`) plus the optional full `UIUC_All` set.
* **`system_files/download_airfoils.py` / `download_xfoil.py`**: One-shot helpers to (re)fetch the full UIUC airfoil database and the XFOIL binary.
* **`system_files/rapid_config.py`**: Central configuration schema + loader (single source of truth for all tunable defaults).
* **`system_files/rapid_config.json`**: Active configuration written by `CONFIGURE.bat` and read by `RUN_OPTIMIZATION.bat`.
* **`system_files/bwb_config.json`**: Ready-made BWB/tailless preset used by `RUN_BWB.bat`.
* **`airfoil_database/Reflexed/`**: Curated reflexed / tailless airfoil catalog (MH, Eppler, EH, S-series) used by BWB co-optimization.

## ⚙️ First-Time Setup
Install Python dependencies (`pip install -r requirements.txt`), place the OpenVSP 3.47.0 win64 build under `system_files/OpenVSP-3.47.0-win64/`, and fetch XFOIL with `download_xfoil.py`. Full step-by-step instructions are in **[SETUP.md](SETUP.md)**.

The curated airfoil catalogs (`Low_Re` / `Medium_Re` / `High_Re` / `Reflexed`) ship with the repo. `SETUP.bat` also fetches the full ~1650-airfoil UIUC set (`download_airfoils.py`), which is recommended — the airfoil co-optimization can sweep the whole database for the best section per station.

## 📖 Usage Instructions

1. Run **`RUN_OPTIMIZATION.bat`**.
2. Enter the number of CPU cores to allocate (Default: 32). The engine runs fully in parallel utilizing a dynamic worker pool.
3. Enter the **Wake Iteration** count.
4. The system will automatically detect existing checkpoints. If a checkpoint is found, it asks for the number of continuation generations. If not, it initializes a fresh run by asking for population size and generation count.
5. You can stop the process at any point using **`Ctrl + C`**. The current generation's analysis will cleanly finish and save the state, allowing you to resume exactly where you left off later.

* **Performance benchmark:** With standard settings (32 cores, 32 pop size, 10 generations, 0 wake iterations), evaluation takes roughly 30 seconds per generation (~4 minutes total) on a 16 physical / 32 logical core machine.
* **Recommendation:** It is highly recommended to run for at least 100 generations to allow the Pareto Front to mature.
* **Physics Note:** `0 wake iterations` solves with a **fixed wake**. This means downwash interactions are neglected but the solver becomes highly stable and extremely fast. If a tailplane is introduced in future steps, wake iterations should be increased.

### 🔄 Resetting the Run
To completely wipe the `wing_models/` directory and reset the optimization history, run **`RESET.bat`**. As a safety measure, it will prompt you for confirmation (`y`) three times before proceeding.
* *Note: The current architecture is highly lightweight, so resetting will not yield massive disk space recovery, but serves as a clean slate.*

## ✈️ Airfoil Co-Optimization *(Experimental)*
After the planform Pareto front matures, the optimizer can chain into an **airfoil co-optimization stage** that tailors the sectional profiles to the winning wing:

1. **Sectional extraction:** Local chord, Reynolds number, and effective angle of attack are extracted at the root, kink, and tip stations of the selected wing.
2. **Parallel XFOIL sweep:** Candidate airfoils from the relevant Reynolds regime catalog are screened in parallel through the embedded XFOIL solver; thin/structurally weak sections are rejected and unphysical solver artifacts are purged.
3. **Geometric refinement (optional):** The winning profiles can be auto-refined via XFOIL `TFAC` thickness/camber scaling to squeeze out additional L/D.
4. **Back-feed:** The best profile per station is written to `system_files/active_airfoils.json` and deployed into `system_files/active_airfoils/`. Subsequent OpenVSP wing models are instantiated with these custom `.dat` profiles instead of the baseline NACA section.

By default `active_airfoils.json` points all three stations at the baseline `naca2412`, so a fresh clone runs the pure-planform path until the co-optimization stage overwrites it. This stage is at an early/experimental maturity.

## 🏷️ Wing Naming Convention
Optimal solutions are dynamically named based on their geometric parameters:
`W(AR)/(Span)/(Kink_Frac)/(Sweep_Inner)-(Sweep_Outer)/(Taper_Inner)-(Taper_Outer)`

* **Example:** `W91/39/5/7-5/6-8` represents:
  * **Aspect Ratio (AR):** 9.1
  * **Span:** 39 m
  * **Kink Fraction:** 0.5
  * **Sweep (Inner / Outer):** 7° / 5°
  * **Taper (Inner / Outer):** 0.6 / 0.8 (Decimals scaled by 10)

## 🛠️ Configuring Constraints & Parameters
Run **`CONFIGURE.bat`** to edit the study without touching source code. It walks through every tunable parameter and, for each one, shows both the built-in **default** and the value you entered **last** time — press Enter to keep the current value or type a new one.

Editable groups:

* **Constraints:** minimum L/D, max |Cm| (conventional mode), minimum CL.
* **Design bounds:** lower/upper limits for aspect ratio, kink fraction, inner/outer taper, and inner/outer sweep.
* **Flight conditions:** true airspeed, air density, temperature, viscosity, reference area, angle of attack, root incidence.
* **BWB settings:** static-margin window, trim tolerance, airfoil catalog, CG location.

Values are saved to `system_files/rapid_config.json` and applied automatically on the next `RUN_OPTIMIZATION.bat`. If the file is absent, the built-in defaults are used, so the tool always runs.

## 🦅 BWB / Tailless Optimization
Run **`RUN_BWB.bat`** to optimize a blended-wing-body / tailless configuration. It points the engine at the `system_files/bwb_config.json` preset (higher sweep, lower taper, `BWB_MODE` on) and changes the physics in three ways:

1. **Longitudinal stability as a hard constraint.** Because there is no tail, each candidate is solved at **two angles of attack** in the same VSPAERO run and the pitching-moment slope is measured. The static margin is computed as `SM = −dCm/dCL` (about the CG set by `X_cg`, defaulting to 0.25·MAC) and constrained to the `SM_MIN … SM_MAX` window.
2. **Trim as a hard constraint.** The pitching moment at the design point must satisfy `|Cm| ≤ CM_TRIM_TOL`, so the wing can trim itself without an elevator. (In BWB mode the conventional `CM_MAX` cap is replaced by these stability/trim constraints.)
3. **Reflexed airfoils.** The co-optimization sweep draws from the `airfoil_database/Reflexed/` catalog and the winner selection rewards near-zero / slightly-positive `Cm` (reflexed sections) rather than pure L/D, so the chosen profiles reinforce trim instead of fighting it.

Stability and aerodynamics are evaluated **simultaneously** (one VSPAERO run yields CL, L/D, Cm and the moment slope together), so the static-margin and trim requirements drive the planform search directly rather than being checked after the fact. Tune the window and bounds in `system_files/bwb_config.json` (or copy values in with `CONFIGURE.bat`).

> **Note:** The static margin here is a low-to-medium fidelity estimate from a two-point linear `dCm/dCL`. Set `CG_X` to your intended CG for a meaningful margin, and validate promising tailless designs with higher-fidelity tools.

## 📊 Visualizations & Plotting

* Run **`RUN_PLOTTER.bat`** or **`RUN_PERFORMANCE_PLOTTER.bat`** to visualize Pareto sets and planform designs.
* Graphics are saved sequentially as `PLOTS/performance_plot_X.png` without overwriting existing files.
* **Color Legend:**
  * 🟢 **Green points:** Feasible designs that fully satisfy constraints.
  * 🔴 **Red points:** Violating designs (excluded from Pareto considerations).
* **Interactive Popups:** When launched, a high-performance UI pops up. Moving your mouse over the wing subplots instantly highlights their respective Pareto coordinate (scaling up to 4x and glowing golden), and vice versa.
* **Performance Map:** In addition to the standard $C_L$ vs. $L/D$ charts, the tool features a specialized propeller aircraft performance map plotting **Range Efficiency ($L/D$)** against **Endurance Efficiency ($C_L^{1.5}/C_D$)**.

*A pre-computed sample run consisting of 100 generations is readily available inside the `PLOTS` directory.*

## ⚖️ Objectives and Constraints
* **Objectives:** Maximize $L/D$ (Aerodynamic Efficiency) and $C_L$ (Lift Coefficient).
* **Constraints:** 
  * $L/D \geq 15.0$
  * $C_M \leq 0.35$ (Absolute longitudinal stability limitation)
  * $C_L \geq 0.15$
* Upper bounds for $C_L$ and $L/D$ are also actively imposed to filter out non-physical diverging solver results.

## 📈 Live Process Monitoring *(Currently Disabled)*
* **`n_gen`**: Current generation count.
* **`n_eval`**: Total number of unique wing designs evaluated.
* **`n_nds`**: "Non-dominated Solutions" count (number of elite candidates sitting on the active Pareto Front).
* **`cv_min` / `cv_avg`**: Constraint Violation metrics. Convergence toward `0` indicates feasible populations.
* **`eps`** (Epsilon): Measures progression rate between consecutive generations.
* **`indicator`**: Standard multi-objective metrics monitoring algorithm convergence quality.

##   Future Roadmap

### 1. Coupled Wing-Tail Design
Simultaneously designing the wing and the tailplane. This will enable integrated, co-optimized planform sizing alongside trim and longitudinal static margin assessments.

### 2. Iterative Airfoil-Planform Co-Optimization ✅ *Implemented (experimental)*
XFOIL is now integrated directly into the workflow (see **Airfoil Co-Optimization** above): local sectional properties are extracted from the converged planform, candidate airfoils are screened in parallel through XFOIL, and the winners are back-fed into OpenVSP. Remaining work is to tighten the loop into a fully automated, multi-pass planform↔airfoil convergence cycle.

### 3. Discrete Airfoil Databases & Scraping ✅ *Implemented (experimental)*
The full UIUC airfoil database is fetched via `download_airfoils.py` and curated into Reynolds-regime catalogs. XFOIL batch evaluations build local polar lookups, candidate pools are screened per station, and root/kink/tip profiles are assigned independently. Future work: expose "Airfoil Type" as a first-class discrete design variable inside the NSGA-II search itself.
